Markdown 基本语法

Markdown 标题

在Markdown中，创建标题是在标题前面添加井号（"#"）。 #的数量代表了标题的级别。例如，添加三个# 表示创建一个三级标题（<h3>）
例如：### 我是三级标题

Markdown HTML 效果
# 我是标题 <h1>我是标题</h1>

我是标题
## 我是标题 <h2>我是标题</h2>

我是标题
### 我是标题 <h3>我是标题</h3>

我是标题
#### 我是标题 <h4>我是标题</h4>

我是标题
##### 我是标题 <h5>我是标题</h5>
我是标题
###### 我是标题 <h6>我是标题</h6>
我是标题

或者...

还可以在标题下方输入任意数量的 == 号来标识一级标题， 或用 -- 来标识二级标题。

Markdown HTML 效果
我是标题
==========		 <h1>我是标题</h1>

我是标题
我是标题
------------------		 <h2>我是标题</h2>

我是标题

注意

不同的平台处理标题前的 # 需不需要空格的方式不一致。 最好使用一个空格在 # 与标题之间进行分隔。

✅ 正确示范 ❌ 错误示范
# 这是标题 #这是标题




Markdown中的段落

一般使用空白行来分割段落，比如说：

Markdown HTML 效果
我是一个段落

我也是一个段落		 <p>我是一个段落</p>

<p>我也是一个段落</p>		 我是一个段落

我也是一个段落

注意

不要用空格或制表符（Tab）缩进段落。

✅ 正确示范 ❌ 错误示范
我是一个段落

我也是一个段落		   我是一个段落

    我也是一个段落




Markdown中的换行

在一行末尾添加2个或2个以上的空格，然后按回车键，即可换行（或使用<br>）

Markdown HTML 效果
我是一行文字。
我也是一行文字。		 <p>我是一行文字。<br>
我也是一行文字</p>		 我是一行文字。
我也是一行文字。

注意

几乎每个 Markdown 应用程序都支持两个或多个空格进行换行，称为 结尾空格（trailing whitespace）。 但最好使用其他方式，因为很难在编辑器中直接看到空格。幸运的是，几乎每个 Markdown 应用程序都支持另一种换行方式：HTML 的 <br> 标签。

为了方便看见或兼容性，最好在行尾添加“结尾空格”或 HTML 的 <br> 标签来换行。

还有两种其他方式我并不推荐使用。CommonMark 和其它几种轻量级标记语言支持在行尾添加反斜杠( \ ) 的方式进行换行，但是并非所有 Markdown 应用程序都支持此种方式，因此从兼容性的角度来看，不推荐使用。 至少有两种轻量级标记语言支持无须在行尾添加任何内容，只须键入回车键即可实现换行。

✅ 正确示范 ❌ 错误示范
使用两个空格来换行。  
我是下一行。

使用 HTML 中的 <br> 标签来换行。<br>
我是下一行。 		使用反斜杠来换行。\
我是下一行。

直接按回车来换行。
我是下一行。




Markdown中的强调语法

通过将文本设置为粗体或斜体来强调其重要性。


粗体（Bold）

要加粗文本，一般在要加粗文本的前后各添加两个星号（asterisks）或两个下划线（underscores）。 如果想要加粗的文本在其他文本的中间部分用于强调时，请在要加粗部分的两侧各添加两个星号或两个下划线。

Markdown HTML 效果
我是文字，**我是加粗部分** 我是文字,<strong>我是加粗部分</strong> 我是文字，我是加粗部分
我是文字，__我是加粗部分__ 我是文字,<strong>我是加粗部分</strong> 我是文字，我是加粗部分
我是文字，__我是加粗部分__,我是文字 我是文字,<strong>我是加粗部分</strong>,我是文字 我是文字，我是加粗部分,我是文字

注意

各个平台在处理文字中间的下划线是否加粗并不一致。为兼容考虑，最好使用星号加粗
